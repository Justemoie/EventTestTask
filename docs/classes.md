# Glossary

| ***Term** | **Definition** |
|---|---|
| **User** | Представляет собой физическое лицо, которое использует приложение для регистрации на мероприятия и создания мероприятий.Пользователь может войти, зарегистрироваться, управлять своими мероприятиями и регистрироваться на чужие.   |
| ***Event** | Мероприятие с определенными атрибутами, такими как название, описание, дата начала, дата окончания, локация, категория и максимально число участников. |
| ***RegistrationService** |  Класс-связка, который реализует отношение "многие-ко-многим" между User и Event. Он нужен для реализации дополнительной логики о факте регистрации пользователей на мероприятия. |
| **AuthenticationService** | Класс услуг, который обрабатывает аутентификацию пользователя, включая функции входа в систему, регистрации и выхода из системы. Он проверяет учетные данные пользователя и управляет пользовательскими сеансами.  |
| **EventService** | Класс, управляющий коллекцией мероприятий. Он предоставляет методы для отображения доступных мероприятий, их фильтрация на основе критериев и сортировка по предпочтениям пользователя. |
